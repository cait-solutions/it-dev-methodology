# ADR — Architecture Decision Records

Каталог архитектурных решений methodology-platform.

ADR — записи фиксирующие **почему** принято архитектурное решение, какие альтернативы рассматривались, и какие последствия. Не заменяет SYSTEM-MAP (структура) и не заменяет CLAUDE.md (правила).

---

## Когда создавать ADR

- Принято решение влияющее на больше чем один компонент.
- Выбран один из нескольких архитектурных вариантов.
- Зафиксирован контракт между сервисами / модулями.
- Решение требует обоснования для будущих разработчиков ("почему так, а не иначе").

## Когда НЕ создавать ADR

- Деталь реализации алгоритма (это design spec в `docs/services/...`).
- Стилистическое решение (это `.claude/rules/*.md`).
- Локальная задача (это план).

---

## Процесс

1. Скопируй [`_TEMPLATE.md`](./_TEMPLATE.md) → `ADR-NNN-short-english-slug.md`.
2. Имя файла — только английский. Содержимое — язык проекта.
3. Заполни поля: Статус / Контекст / Решения / Последствия.
4. Под-решения с явным статусом РЕШЕНО или ОТКРЫТО.
5. Открытые вопросы → создай OQ-NNN в `OPEN-QUESTIONS.md`.
6. Добавь строку в таблицу ниже.
7. После реализации — обнови статус ADR в том же PR что и код.

## Соглашения

- **Имя файла:** `ADR-NNN-short-english-slug.md`
- **Нумерация:** последовательная, никогда не переиспользуется
- **Статусы:**
  - **Открыт** — обсуждается, решение не принято
  - **Принят** — решение зафиксировано, реализация впереди
  - **Частично реализован** — часть подразделов внедрена
  - **Реализован** — всё внедрено, ADR работает как документация
  - **Заменён ADR-NNN** — supersedes pattern, не редактируем старый ADR при изменении подхода
- **Supersedes pattern:** при пересмотре решения создаётся новый ADR со ссылкой на старый. Старый помечается `Заменён ADR-NNN`, но не удаляется.

---

## Связь с другими артефактами

- **SYSTEM-MAP.md** — структура (что есть). ADR — почему именно так.
- **CLAUDE.md** — правила для агентов (как кодить). ADR — обоснование правил.
- **VISION** — стратегия (куда идём). ADR — конкретные шаги.
- **RISKS.md** — что может пойти не так. ADR может закрывать риск.
- **OPEN-QUESTIONS.md** — открытые вопросы из ADR живут здесь.
