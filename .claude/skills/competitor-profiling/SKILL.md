# competitor-profiling — Анализ конкурентов

**Порядок Foundation block:** третий (после `define-positioning` и `customer-research`). Читает Positioning и ICP из MARKETING.md чтобы критерии сравнения соответствовали целевому сегменту.

Ты помогаешь провести детальный анализ конкурентной среды: составить профили конкурентов, выявить их позиционирование, найти незакрытые боли клиентов и определить конкурентные преимущества продукта. Работаешь строго в рамках маркетинг-контекста проекта.

**Артефакт:** `MARKETING.md` (корень проекта) — секция `## Competitor Profiles`

---

## Шаг 1 — Загрузка контекста

1. Прочитать `MARKETING.md` (если существует) — все секции
2. Обратить особое внимание на:
   - `## Positioning` → секция `### Key Alternatives` — список конкурентов из позиционирования
   - `## ICP & Personas` → `### Anti-ICP` и `### JTBD` — что важно клиентам при выборе
3. Прочитать `PRODUCT.md` — что продукт обещает пользователям
4. Если `MARKETING.md` не существует:
   ```
   MARKETING.md не найден. Создать? (y/n)
   При y — создаю из шаблона и заполняю секцию Competitor Profiles.
   ```

---

## Шаг 2 — Определить список конкурентов

На основе загруженного контекста составить список:

**Категории конкурентов:**

| Категория | Описание | Пример |
|---|---|---|
| **Direct** | Решают ту же проблему для того же сегмента | Прямые аналоги |
| **Indirect** | Решают ту же проблему иначе | Другой класс решений |
| **Substitute** | Клиент мог выбрать вместо покупки | Excel, найм, ручной процесс |
| **Future** | Ещё не конкурент, но движется в вашу сторону | Крупный игрок расширяющий продукт |

Попросить пользователя подтвердить список или добавить конкурентов которых не хватает. Не профилировать более 6-8 конкурентов за раз — глубина важнее широты.

---

## Шаг 3 — Clarifying questions (если контекст недостаточен)

Задай только вопросы ответы на которые не найдены в загруженном контексте:

1. **Кого клиенты называют альтернативой?** (из интервью, отзывов, sales-разговоров)
2. **Кого ты видишь в тендерах или deal cycles?** (с кем конкурируешь чаще всего)
3. **Кто потерял клиентов которых ты выиграл?** и наоборот

---

## Шаг 4 — Профиль каждого конкурента

Для каждого конкурента из списка заполни профиль:

### Шаблон профиля

```
### [Название конкурента]

**Категория:** direct / indirect / substitute / future
**Сайт:** [URL если известен]
**Основан / Раунд:** [год основания / стадия финансирования если известно]

#### Позиционирование
**Как они себя позиционируют:** [их собственный messaging — с сайта или материалов]
**Целевой сегмент:** [кому они продают]
**Ключевое обещание:** [их главный value prop]

#### Продукт
**Ключевые features:** [что входит в основной продукт]
**Ценообразование:** [модель — freemium / subscription / usage-based / enterprise; ценовой диапазон если известен]
**Интеграции:** [с чем интегрируются]

#### Сильные стороны
- [Сильная сторона 1 — конкретно, не "хороший UX"]
- [Сильная сторона 2]
- [Сильная сторона 3]

#### Слабые стороны / Gap-ы
- [Слабость 1 — подтверждённая из отзывов или интервью]
- [Слабость 2]

#### Кто их выбирает
[Сегмент который предпочитает их — и почему]

#### Кто НЕ выбирает их (и почему)
[Сегмент который уходит от них или не рассматривает — это наша возможность]

#### Наш ответ
**Где мы сильнее:** [конкретно, с обоснованием]
**Где они сильнее:** [честно — это защищает от слабых аргументов в sales]
**Battlecard тезис:** [одно предложение — почему выбирают нас, а не их]

#### Источники данных
- [ ] Сайт / материалы
- [ ] Отзывы (G2, Capterra, App Store)
- [ ] Интервью с клиентами которые пробовали
- [ ] Публичные кейсы / PR
- [ ] Конкурентная разведка (Glassdoor, LinkedIn, job postings)
```

## Шаг 8 — Запись в MARKETING.md

После заполнения:

1. Показать черновик секции `## Competitor Profiles` пользователю
2. Спросить: `Записать в MARKETING.md? (y/n/edit)`
3. При `y` — записать/обновить секцию в файл. **Важно:** обновить только секцию `## Competitor Profiles`, не трогать `## Positioning` и `## ICP & Personas`
4. При добавлении нового конкурента к существующим — **append профиль** в секцию, не перезаписывать секцию целиком
5. При `edit` — дать возможность внести правки перед записью
6. При `n` — сохранить в памяти сессии, не записывать

**Формат секции в MARKETING.md:**

```markdown
## Competitor Profiles

_Обновлено: YYYY-MM-DD_

### [Название конкурента]
**Категория:** [direct / indirect / substitute / future]
**Позиционирование:** [их messaging]
**Сильные стороны:** [список]
**Слабые стороны:** [список]
**Кто их выбирает:** [сегмент]
**Наш ответ:** [battlecard тезис]
**Источники:** [список]

### [Следующий конкурент]
...

### Конкурентная матрица
[таблица из Шага 5]

### Gap-ы (возможности)
[таблица из Шага 6]
```
