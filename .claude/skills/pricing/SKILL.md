## Основы ценообразования

### Три оси ценообразования

**1. Пакетирование** — Что включено на каждом тарифе?
- Функции, лимиты, уровень поддержки
- Как тарифы отличаются друг от друга

**2. Метрика ценообразования** — За что берём деньги?
- Per user, per usage, flat fee
- Как цена масштабируется с ценностью

**3. Прайс-поинт** — Сколько берём?
- Фактические суммы
- Воспринимаемая ценность vs стоимость

### Value-Based ценообразование

Цена должна основываться на доставляемой ценности, не на стоимости обслуживания:

- **Воспринимаемая ценность клиента** — Потолок
- **Ваша цена** — Между альтернативами и воспринимаемой ценностью
- **Лучшая следующая альтернатива** — Пол для дифференциации
- **Стоимость обслуживания** — Только baseline, не основа

**Ключевой инсайт:** Цена между следующей лучшей альтернативой и воспринимаемой ценностью.

## Метрики ценообразования

### Что такое метрика ценообразования?

Метрика ценообразования — это то за что берёшь деньги. Должна масштабироваться с ценностью которую получают клиенты.

**Хорошие метрики:**
- Выравнивают цену с доставляемой ценностью
- Легко понять
- Масштабируются по мере роста клиента
- Сложно обойти

### Распространённые метрики

| Метрика | Лучше всего для | Пример |
|---------|-----------------|--------|
| Per user/seat | Инструменты для совместной работы | Slack, Notion |
| Per usage | Переменное потребление | AWS, Twilio |
| Per feature | Модульные продукты | HubSpot add-ons |
| Per contact/record | CRM, email инструменты | Mailchimp |
| Per transaction | Платежи, маркетплейсы | Stripe |
| Flat fee | Простые продукты | Basecamp |

### Выбор метрики

Спроси: "По мере роста использования [метрики] клиентом — получает ли он больше ценности?"
- Если да → хорошая метрика
- Если нет → цена не выровнена с ценностью

## Структура тарифов

### Фреймворк Good-Better-Best

**Good (Базовый):** Ключевые функции, ограниченное использование, низкая цена
**Better (Рекомендуемый):** Полные функции, разумные лимиты, anchor цена
**Best (Премиум):** Всё, расширенные функции, 2-3x цена Better

### Дифференциация тарифов

- **Feature gating** — Базовые vs расширенные функции
- **Usage limits** — Те же функции, разные лимиты
- **Уровень поддержки** — Email → Приоритет → Выделенный
- **Доступ** — API, SSO, кастомный брендинг

## Исследование ценообразования

### Метод Van Westendorp

Четыре вопроса определяющие приемлемый диапазон цен:
1. Слишком дорого (не рассматривали бы)
2. Слишком дёшево (сомнения в качестве)
3. Дорого но могли бы рассмотреть
4. Выгодно

Анализ пересечений для нахождения оптимальной ценовой зоны.

## Когда повышать цены

### Сигналы что пора

**Рыночные сигналы:**
- Конкуренты подняли цены
- Потенциальные клиенты не реагируют на цену
- "Это так дёшево!" обратная связь

**Бизнес-сигналы:**
- Очень высокие conversion rates (>40%)
- Очень низкий churn (<3% ежемесячно)
- Сильная unit economics

**Продуктовые сигналы:**
- Значительно добавлена ценность с момента последнего ценообразования
- Продукт более зрелый/стабильный

### Стратегии повышения цен

1. **Grandfather существующих** — Новая цена только для новых клиентов
2. **Отложенное повышение** — Объявить за 3-6 месяцев
3. **Привязать к ценности** — Поднять цену но добавить функции
4. **Реструктуризация планов** — Полностью изменить планы

## Лучшие практики для страницы цен

### Выше сгиба
- Чёткая таблица сравнения тарифов
- Выделен рекомендуемый тариф
- Переключатель месячная/годовая
- Основной CTA для каждого тарифа

### Типичные элементы
- Таблица сравнения функций
- Для кого каждый тариф
- Секция FAQ
- Выноска годовой скидки (17-20%)
- Гарантия возврата денег
- Логотипы клиентов/знаки доверия

### Психология ценообразования
- **Якорение:** Показывать более дорогой вариант первым
- **Эффект приманки:** Средний тариф должен быть лучшим по соотношению ценность/цена
- **Charm pricing:** $49 vs $50 (для value-ориентированных)
- **Круглые цены:** $50 vs $49 (для премиума)
