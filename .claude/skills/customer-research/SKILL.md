## Шаг 3 — Clarifying questions (если контекст недостаточен)

Задай только те вопросы, ответы на которые не найдены в загруженном контексте:

1. **Кто сейчас покупает продукт?** Роль, индустрия, размер компании (если B2B) / демо (если B2C)
2. **Что происходило в жизни клиента ДО покупки?** Триггер — событие которое запустило поиск решения
3. **Какую задачу они пытались решить?** Функциональную / эмоциональную / социальную
4. **Что они пробовали раньше?** Альтернативы которые не сработали
5. **Что для них "успех"?** Конкретный результат через 30/90 дней использования

Не задавай все 5 сразу. Максимум 2-3 вопроса за раз.

## Шаг 4 — ICP Framework

### 4.1 Primary ICP

Заполни на основе данных из Шага 2-3:

**Профиль (B2B):**
- **Роль / Title:** кто принимает решение или является основным пользователем
- **Индустрия:** сектор, тип бизнеса
- **Размер компании:** ARR, headcount, стадия (seed/series A/enterprise)
- **Гео:** если применимо

**Профиль (B2C):**
- **Демография:** возраст, занятость, доход (если релевантно)
- **Контекст жизни:** в какой ситуации находится покупатель
- **Технологический профиль:** как использует технологии

**Психографика (и B2B, и B2C):**
- **Главная боль прямо сейчас:** что не даёт спать ночью
- **Ценности:** что важно при принятии решений (скорость, надёжность, цена, статус)
- **Страхи:** чего боится при выборе нового инструмента

**Покупательское поведение:**
- **Триггер покупки:** событие которое запустило поиск решения
- **Процесс принятия решения:** самостоятельно / комитет / influencer
- **Критерии выбора:** по чему сравнивают альтернативы
- **Барьеры:** что мешает купить сейчас

### 4.2 Secondary ICP (если данные указывают на второй сегмент)

Аналогичный формат. Указать: почему это отдельный сегмент, не вариация Primary ICP.

### 4.3 Anti-ICP (кого НЕ берём)

Явно назвать сегменты которые:
- Создают высокую нагрузку на поддержку без соответствующей прибыли
- Требуют кастомизации противоречащей продуктовой стратегии
- Имеют несовместимые ценности с продуктом

## Шаг 5 — JTBD (Jobs-to-be-done)

Для каждого ICP — сформулировать 3-5 Jobs:

**Формат:** `Когда [ситуация], я хочу [мотивация], чтобы [ожидаемый результат]`

Три типа jobs:
- **Функциональный:** что клиент хочет сделать практически
- **Эмоциональный:** как хочет себя чувствовать в процессе
- **Социальный:** как хочет выглядеть в глазах других

**Пример:**
- Функциональный: `Когда начинаю новый проект с AI-агентом, я хочу иметь готовую структуру команд и правил, чтобы не тратить недели на настройку с нуля`
- Эмоциональный: `Когда деплою изменения, я хочу быть уверен что ничего не сломаю, чтобы не бояться каждого коммита`

## Шаг 6 — Персоны (опционально, если есть данные)

Персона = конкретный представитель ICP с именем и историей. Используй реальные цитаты из интервью/отзывов.

**Формат персоны:**

```
### [Имя] — [Роль / Архетип]

**Профиль:** [1-2 предложения — кто этот человек]

**Цель:** [что хочет достичь с помощью продукта]

**Боли:**
- [Боль 1]
- [Боль 2]

**Цитата:** "[прямая цитата из интервью или отзыва]"

**Сценарий использования:** [конкретная ситуация когда использует продукт]
```

Создавай персону только если есть достаточно данных (минимум 3-5 источника для одной персоны). Не выдумывай детали — лучше оставить поля пустыми с пометкой `[данных нет]`.

## Шаг 7 — Review Mining (если есть отзывы)

Если пользователь предоставил отзывы — провести анализ по паттернам:

**Категории для кодирования:**
- 🟢 Ценность — за что хвалят, что работает
- 🔴 Боль — что не нравится, что не работает
- 🟡 Comparison — с чем сравнивают, кого называют альтернативой
- 💡 JTBD — какую работу пытались выполнить

**Формат анализа:**
```
Проанализировано отзывов: N
Топ-3 ценности: [список с частотой]
Топ-3 боли: [список с частотой]
Названные альтернативы: [список]
Ключевые JTBD цитаты: [3-5 цитат]
```
