## Правила работы

- **Каждая мера защиты должна быть привязана к угрозе** — security theater недопустим.
- **OWASP Top 10 в контексте проекта** — не общий чек-лист, а конкретные векторы.
- **Не пиши код, не запускай тесты** — только статический анализ + WebFetch для проверки CVE.

## Чеклист проверки

### Аутентификация и авторизация

- [ ] Новый endpoint требует аутентификации? (или это публичный по дизайну?)
- [ ] Tenant / org / user context извлекается из надёжного источника (JWT, signed token), не из request header без проверки?
- [ ] Нет endpoint-ов возвращающих данные другого tenant / user?
- [ ] Privilege escalation невозможен через payload manipulation (mass assignment, type confusion)?

### PII и чувствительные данные

- [ ] API возвращающий PII — есть политика доступа?
- [ ] Логи не содержат PII / токены / ключи?
- [ ] Error responses не утекают внутренние детали (stack traces, паттерны запросов)?
- [ ] API-ключи внешних сервисов — в secret store, не в коде / миграциях / commit'ах?

### Внешние интеграции

- [ ] Timeout и retry настроены?
- [ ] Ответы от внешних API валидируются перед использованием?
- [ ] Нет open redirect / SSRF через пользовательские URL?
- [ ] Webhook signatures проверяются (если применимо)?

### OWASP Top 10 в контексте

#### Injection

- [ ] SQL — параметризованные запросы / ORM, нет raw SQL с user input?
- [ ] Command injection — нет shell с user input?
- [ ] XSS — output escaping на границе render?

#### Broken Access Control

- [ ] IDOR — каждый запрос проверяет ownership / access?
- [ ] Force browsing — non-public endpoints требуют auth?

#### Cryptographic Failures

- [ ] Чувствительные данные шифруются at rest где применимо?
- [ ] HTTPS обязателен для внешних точек входа?

#### Other (адаптировать под стек)

- [ ] <stack-specific OWASP проверки>

## Security theater — что флагать

Если предлагаемая мера не имеет ни одного из:
1. Конкретная угроза из CLAUDE.md против которой работает
2. Способ проверить что мера реально работает (тест, не "установлена")
3. Понимание остаточных рисков

— это security theater. Флагать как 🟡 с пометкой "обосновать угрозу или не делать".

Типичные симптомы:
- "Установить X" без конфигурации под конкретную угрозу
- "Закрыть порт Y" без понимания через что реально атакуют
- "Default deny outgoing" без полного списка зависимостей приложения
