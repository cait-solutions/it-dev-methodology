# IT Dev Methodology

AI-assisted разработка: slash-команды, шаблоны, хуки, скрипты — единый источник правды для всех проектов.

---

## Для нового разработчика — начни здесь

Проект уже создан. Тебе нужно развернуть его на своей машине.

### Шаг 1. Клонируй репозитории

Создай папку-контейнер для проекта, запроси у PM ссылки на нужные репозитории и склонируй их туда.

Всего будет минимум три репозитория рядом в одной папке:
- **it-dev-methodology** — этот репо, методология. Публичный — клонируется без токена, [смотри видео](https://flm.by/vbpx4u6vnJY)
- **project-documentation** — артефакты, архитектура, команды агента
- **project-backend**, **project-frontend**, ... — репозитории с кодом (их может быть несколько, по одному на каждый сервис)

Точные названия репо узнай у PM.

> Для клона приватных репозиториев установи [GitHub CLI](https://cli.github.com) и выполни в терминале:
> ```bash
> gh auth login
> ```
> В интерактивном меню выбери: **GitHub.com → HTTPS → Login with a web browser**. В терминале появится одноразовый код — скопируй его, нажми Enter. Откроется браузер, вставь код в поля и подтверди вход. Код приходит не на почту — он отображается только в терминале. [Смотри видео](https://flm.by/vbpnESfibne)
>
> **Важно:** разные репозитории могут быть доступны под разными GitHub аккаунтами. Если при клоне появляется ошибка `repository not found` — скорее всего активен не тот аккаунт. Сначала добавь нужный аккаунт через `gh auth login` (если ещё не добавлен), затем переключись:
> ```bash
> gh auth switch
> ```
> Команда покажет список всех добавленных аккаунтов — выбери нужный и повтори клон.

### Шаг 2. Настрой workspace в IDE

Установи расширение Claude Code для [VS Code](https://marketplace.visualstudio.com/items?itemName=Anthropic.claude-code) или [JetBrains](https://plugins.jetbrains.com/plugin/24819-claude-code).

Открой папку `project-documentation/` как основную рабочую папку в IDE, затем добавь остальные репозитории через **Add Folder to Workspace** — так все репо будут видны агенту одновременно.

Сохрани настройки workspace в папку-контейнер (**Save Workspace As...**) — в следующий раз достаточно открыть этот файл и всё подключится автоматически.

[Видео: настройка workspace в VS Code](https://flm.by/vbp57eVjeJT) · [Видео: Add Folder to Workspace](https://flm.by/vbpc1q1QQv2)

### Шаг 3. Добавь токены доступа

Для работы с приватными репозиториями агенту нужны токены (Fine-grained PAT) — они позволяют пушить коммиты от твоего имени. Получи токены у PM или создай сам в настройках GitHub каждого репо.

В Claude Code запусти:

```
/secrets
```

Команда покажет все доступные операции. Для добавления нового токена выбери **Добавить новый секрет** и следуй инструкции — значение вводится в терминале, агент его не видит.

> **Важно:** никогда не вводи токены напрямую в чат с агентом. Всё что написано в чате попадает в историю сессии и может утечь. Система secrets специально устроена так, что агент получает токен только через защищённый механизм — он его использует но никогда не видит значение.

### Шаг 4. Онбординг

В Claude Code запусти:

```
/onboard
```

Команда покажет архитектуру проекта и что делать дальше.

## Обновление методологии

Обновления доставляются **push-only**: maintainer методологии запускает `/push-consumers`
с репозитория методологии — консьюмеры не обновляются сами. Проверить здоровье install
(read-only healthcheck версии / хуков / секретов):

```
bash scripts/sync-doctor.sh
```
