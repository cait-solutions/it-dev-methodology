# BEHAVIOR — {{Project Name}}

Поведенческие соглашения и edge cases.

> **Отличие от других артефактов:**
> - **ADR** отвечает "какое архитектурное решение принято".
> - **CLAUDE.md** отвечает "какие правила для разработки".
> - **PRODUCT.md** отвечает "что делает система с точки зрения пользователя в happy path".
> - **BEHAVIOR.md** отвечает "как система ведёт себя в edge cases" — неписаные правила, которые принципиальны но не очевидны.

Заполняй при появлении нестандартных кейсов, обнаруженных в продакшене или при ревью.

---

## Когда добавлять запись

- Поведение в edge case не очевидно из кода.
- Поведение в edge case нужно объяснить новому разработчику чтобы он не "исправил" его как баг.
- Поведение задокументировано в PR-description но потеряется в git history.

## Когда НЕ добавлять

- Очевидное поведение (читается из кода) → не дублируй.
- Архитектурное решение → ADR.
- Баг → DEVLOG `[fix:X]`.
