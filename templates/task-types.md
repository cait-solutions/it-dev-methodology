## Ось 1 — Тип задачи

| Категория | Описание (универсальное) | Доп. шаги |
|---|---|---|
| `[code]`     | Изменение поведения системы              | стандартный набор |
| `[product]`  | Изменение UX / пользовательского опыта   | + продуктовая проверка |
| `[data]`     | Миграции, изменение схемы данных          | + карта данных |
| `[infra]`    | Серверная / воркспейс-конфигурация        | + threat model |
| `[security]` | Защита от угроз                           | + threat model |
| `[process]`  | Изменение процесса / методологии          | упрощённый цикл |
| `[contract]` | Стабильный контракт с потребителем — API, события, формат публикуемого артефакта, публичный интерфейс данных | контракт-анализ обязателен |

## Ось 2 — Режим Lite vs Full

- **Lite:** локальный фикс ≤ 1 артефакта, < 20 строк изменений → сокращённый набор шагов.
- **Full:** всё остальное → все применимые шаги навигационной карты команды.
- При сомнении → Full.

## Применимость по доменам

`[product]` / `[data]` / `[process]` / `[contract]` — кросс-доменные (применимы к marketing,
legal-AI, documentation, lead-gen, e-commerce воркспейсам, не только к software-development).
`[code]` = абстрактное «изменение поведения», НЕ дробится на language-specific варианты.
Dev-only уточнения (code-security regex, unit-test gate) живут в самих командах под пометкой
`dev-domain`, а не в этом ядре.
