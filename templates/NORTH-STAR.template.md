# NORTH-STAR.md — {{Project Name}}

Целевая функция проекта. Декларативный критерий ценности, относительно которого `/roadmap` ранжирует работу по ROI (минимум усилий → максимум вклада в цель).

> **Project-owned, декларативный.** Не синхронизируется автоматически (PRESERVE). Заполняет и обновляет **владелец** — это не live-метрики из CRM/аналитики, а зафиксированная цель + self-reported состояние (Граница 3/11: методология ведёт приоритизацию, не исполняет и не хранит runtime-данные).
> **Читается командой `/roadmap`** как критерий приоритизации. `/roadmap` напоминает обновить состояние если оно устарело.

---

## North Star

Главная метрика ценности этого проекта — то, ради чего он существует. По умолчанию для коммерческого проекта это **рост** (продажи / прибыль / выручка), но критерий **конфигурируем**: выбери метрику, которая реально определяет успех именно этого проекта.

| Поле | Значение |
|---|---|
| **Метрика** | `<например: MRR / число платящих клиентов / лиды в месяц / конверсия / иное>` |
| **Почему именно она** | `<одно предложение — почему эта метрика = успех проекта>` |
| **Единица измерения** | `<€ / шт / % / иное>` |

> Если проект не коммерческий (внутренний инструмент, исследование) — North Star может быть нефинансовым (время до результата, охват, качество). Не подгоняй под «продажи» силой.

## Project role

Роль проекта в портфеле. Определяет **как `/roadmap` считает ценность** работы.

```
project_role: <growth | enabling>
```

- **`growth`** — проект напрямую двигает свою North Star (продаёт, привлекает клиентов, растит выручку). Ценность задачи = вклад в собственную метрику.
- **`enabling`** — обеспечивающий проект (инфраструктура, методология, инструмент), который сам по себе не продаёт, но **ускоряет/разблокирует growth-проекты**. Ценность задачи = leverage на growth-проекты из `dependency-edges` ниже.

> Без честной маркировки enabling-проект рискует «делать ради делать». Enabling оправдан только через leverage на чей-то рост.

### Dependency-edges (только для `enabling`)

Какие growth-проекты обслуживает этот проект. `/roadmap` считает Impact задачи enabling-проекта через её влияние на эти проекты.

| Growth-проект | Как этот проект ему помогает |
|---|---|
| `<имя проекта>` | `<одно предложение — что он разблокирует/ускоряет>` |

> Для `growth`-проекта эту таблицу удалить или оставить пустой.

## Current state (self-reported)

| Поле | Значение |
|---|---|
| **Текущее значение метрики** | `<например: €2.3k MRR>` |
| **Дата обновления состояния** | `<YYYY-MM-DD>` |
| **Краткий комментарий** | `<что произошло с прошлого обновления, опционально>` |

> ⚠️ **Staleness:** `/roadmap` смотрит «Дата обновления состояния». Если она старше порога (по умолчанию ~30 дней) — команда напомнит обновить, прежде чем ранжировать (ранжирование по устаревшему состоянию вводит в заблуждение).

---

## Refresh policy

- **Target** — пересматривать при смене стратегии проекта (редко).
- **Current state** — обновлять при значимом изменении метрики или перед запуском `/roadmap` (cadence: ~ежемесячно или по событию).
- **Project role / dependency-edges** — обновлять если меняется роль проекта в портфеле.
