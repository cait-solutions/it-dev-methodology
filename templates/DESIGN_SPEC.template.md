## Верификация (перед переводом в Final)

> **Запрет финализации:** пока этот раздел не заполнен — статус остаётся `Draft`.
> Заполняется агентом или автором в момент запроса `status: "Final"`.

**Качество требований (§2):**
- [ ] Каждый пункт §2 имеет аргументацию (механизм, не «потому что так лучше»)
- [ ] Каждый пункт §2 имеет пример-сценарий (однострочный, реальный)
- [ ] Слов «возможно», «вероятно», «может быть» нет без OQ-пометки

**Реализуемость:**
- [ ] Source каждого механизма §2: `real code` (file:line) / `ADR-NNN` / `external doc` / `TBD: OQ-N` — не «по памяти»
- [ ] Ни один пункт не противоречит существующим ADR (проверено `git log -1 <adr-file>`)
- [ ] Каждый пример §2.N реально выполним в текущем стеке

**Полнота:**
- [ ] §4 «Не в scope» заполнен явным списком (не «всё остальное»)
- [ ] §6 Open Questions созданы для всего неизвестного — нет скрытых допущений

**Диаграммы:**
- [ ] Каждый нетривиальный §2.N имеет mermaid-диаграмму **или** явную skip-причину
- [ ] Ни один mermaid-блок не содержит `PLACEHOLDER`
- [ ] Выполнен `bash scripts/update-mermaid-links.sh <path>` — ссылки актуальны
- [ ] §8 статус явно указан: `Обязательна` / `Рекомендуется` / `Skip` с причиной

**Результат:** `[ ]` Все пункты OK → переводим в Final / `[ ]` Остались открытые → остаётся Draft
