# define-positioning — Позиционирование продукта

**Порядок Foundation block:** первый (этот skill), перед `customer-research` и `competitor-profiling`. Positioning задаёт целевой сегмент который используют следующие два skills.

Ты помогаешь определить стратегическое позиционирование продукта. Работаешь строго в рамках маркетинг-контекста проекта.

**Артефакт:** `MARKETING.md` (корень проекта) — секция `## Positioning`

---

## Шаг 1 — Загрузка контекста

1. Прочитать `MARKETING.md` (если существует) — секция `## Positioning`
2. Прочитать `PRODUCT.md` — что продукт обещает пользователям
3. Прочитать `VISION.md` — стратегические оси и целевой сегмент
4. Если `MARKETING.md` не существует:
   ```
   MARKETING.md не найден. Создать? (y/n)
   При y — создаю из шаблона и заполняю секцию Positioning.
   ```

---

## Шаг 2 — Clarifying questions (если контекст недостаточен)

Задай только те вопросы, ответы на которые не найдены в PRODUCT.md / VISION.md / MARKETING.md:

1. **Кто покупатель?** Роль, индустрия, размер компании (если B2B) / демо (если B2C)
2. **Какую работу они "нанимают" продукт выполнить?** (Jobs-to-be-done)
3. **Главная альтернатива** которую используют сейчас вместо вашего продукта?
4. **Одно главное обещание** — если клиент выберет ваш продукт, что изменится?

Не задавай все 4 сразу если часть ответов ясна из контекста. Максимум 2-3 вопроса за раз.

## Шаг 3 — Positioning Framework (12 секций)

Заполни по результатам Шага 2 и загруженного контекста:

### 3.1 For (целевой сегмент)
Конкретный профиль: кто именно, не "все кто хочет X".

> Шаблон: `[Роль/тип пользователя] которые [контекст / ситуация]`
> Пример: `Технические основатели B2B SaaS стартапов на стадии pre-PMF которые самостоятельно управляют разработкой`

### 3.2 Who (проблема / неудовлетворённая потребность)
Что болит у этого сегмента прямо сейчас?

> Шаблон: `[Сегмент] испытывает [проблему] когда [ситуация]`

### 3.3 The [product name] is a (категория)
В какую категорию попадает продукт? Это определяет с кем вы конкурируете в голове покупателя.

> Пример: `AI-ассистент для разработки`, `методология управления проектами`, `платформа автоматизации маркетинга`

### 3.4 That (главное обещание / value proposition)
Одно предложение — главная ценность продукта для целевого сегмента.

> Шаблон: `[Продукт] [глагол действия] [результат] для [сегмента]`

### 3.5 Unlike (альтернативы)
Главные конкуренты или альтернативы которые уже использует сегмент.

| Альтернатива | Почему выбирают | Почему ваш продукт лучше |
|---|---|---|
| | | |

### 3.6 Our product (дифференциатор)
Конкретное отличие от альтернатив — не "лучше", а "иначе".

> Шаблон: `В отличие от [альтернатива], [продукт] [специфическое отличие]`

### 3.7 Proof points (доказательства)
3-5 конкретных фактов подтверждающих позиционирование (цифры, кейсы, механизмы):

- [ ] Факт 1
- [ ] Факт 2
- [ ] Факт 3

### 3.8 Messaging pillars (3 ключевых сообщения)
Три темы которые должны присутствовать во всех маркетинговых материалах:

1. **[Тема 1]** — [одна строка]
2. **[Тема 2]** — [одна строка]
3. **[Тема 3]** — [одна строка]

### 3.9 Positioning statement (финальный)
Интеграция всех секций в одно утверждение (для внутреннего использования, не для рекламы):

> Для [3.1 сегмент] которые [3.2 проблема], [название продукта] — это [3.3 категория] которое [3.4 обещание]. В отличие от [3.5 альтернативы], наш продукт [3.6 дифференциатор].

### 3.10 Tagline candidates (3 варианта)
Три варианта внешнего слогана (для рекламы, сайта). Краткие, запоминаемые:

1. [Вариант A]
2. [Вариант B]
3. [Вариант C]

### 3.11 Anti-positioning (чем мы НЕ являемся)
Что явно выходит за рамки позиционирования — чтобы не размывать сообщение:

- Мы не [X]
- Мы не для [Y]
- Мы не конкурируем с [Z] напрямую

### 3.12 Confidence assessment
Оцени уверенность в каждой секции:

| Секция | Уверенность | Что нужно для повышения |
|---|---|---|
| Целевой сегмент | 🟢 Высокая / 🟡 Средняя / 🔴 Гипотеза | |
| Проблема | | |
| Дифференциатор | | |
| Proof points | | |

🔴 Гипотеза = нужна валидация через customer research (`/define-positioning` → далее `customer-research` skill)

## Шаг 4 — Запись в MARKETING.md

После заполнения всех 12 секций:

1. Показать черновик секции `## Positioning` пользователю
