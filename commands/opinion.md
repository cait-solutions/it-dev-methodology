# /opinion — Честное мнение с North Star (Council 7/7)

> **Цель:** получить честное, структурированное мнение агента по конкретному вопросу, заземлённое в North Star проекта. Команда для explicit opinion request — не замена свободному чату, а формализованный путь когда мнение важно для решения.

**Critical Friend Protocol:** не поддакивать, не хеджировать. Чёткая позиция с обоснованием в контексте VISION.md.

**Council Protocol (дефолт):** каждый явный вызов `/opinion` запускает **7 независимых советников** как **external sub-agents** (5 Council-ролей: Ценность · North Star · Горизонт · Скептик · Деятель — плюс Альтернативщик · Complexity tax), каждый со своим **чистым контекстом** → structural independence (нет framing bias, накопленного в диалоге). Главный агент синтезирует их вердикты в Committed Verdict. ~30-60 сек, 7× стоимость — это осознанная цена явного `/opinion`.

> **Зачем external, а не симуляция в одном контексте:** агент, который уже «продал» идею в первых сообщениях диалога, защищает её и в роли Скептика (framing bias). External sub-agent стартует с **чистого slate** → structural independence, не симуляция. Это L4-усиление (отдельный контекст), не L1 (другая инструкция в том же контексте).

**Лёгкий путь — inline `[?]`:** маркер `[?]` в любом тексте → **НЕ** запускает 7 sub-agents. Применяет тот же Opinion Protocol, но **5 советников симулируются в одном контексте** (быстро, ~секунды). Для дешёвого мнения по ходу диалога. Нужен полный council — пиши явный `/opinion`.

> **Почему `[?]` остаётся лёгким (не 7 external):** авто-детект `[?]` срабатывает на реплики по ходу — спавнить 7 sub-agents на каждую = ползучий авто-multi-agent на каждую реплику (нарушает VISION Граница 8 «не fixed multi-agent конвейер»). Лёгкость `[?]` намеренна. Полный council — только на **явный** opt-in `/opinion`.

**`/opinion+` (legacy):** синоним `/opinion`. Поведение council-7 теперь дефолтное для `/opinion`, поэтому отдельный режим `+` больше не нужен — суффикс сохранён только для обратной совместимости (старый ввод не ломается).

## Рекомендуемая модель

**Extended (UI settings):** effort: **High** · thinking: **ON** — синтез 7 вердиктов council = максимальная глубина reasoning. См. `.claude/model-tiers.md` § Effort & Thinking.

**Default tier:** **Capable** (Opus) — синтез 7 независимых external-вердиктов + North Star extraction требуют deep reasoning. Council-7 — дефолт, поэтому Capable обязателен.
**Upgrade:** уже на верхнем tier для команды.
**Downgrade to Default/Fast:** ❌ НЕ рекомендуется для явного `/opinion` — синтез council требует reasoning. (Лёгкий путь `[?]` не запускает команду и pre-flight не триггерит — он исполняется на текущей модели сессии.)
**Mid-task escalation:** нет — opinion-flow не требует reassessment mid-task.
**Pre-flight model check:** да — спроси пользователя о модели при старте, или используй ранее подтверждённую в сессии. При mismatch (текущая < Capable) — рекомендация переключиться перед запуском council.

## Шаг 0 — Pre-flight: обязательные reads

Выполняется **в главном агенте** (sub-agents стартуют с чистого контекста — они НЕ видят эти файлы, если не передать в prompt).

**VISION.md — ОБЯЗАТЕЛЬНЫЙ.** Если не найден → СТОП:

```
⛔ VISION.md не найден.
   Команда /opinion требует VISION.md для North Star anchoring.
   Без него мнение будет generic — не заземлённым в цели проекта.

   Создать VISION.md сейчас? → /vision strategy
```

**doc_repo_path resolution:**
1. Прочитать `CLAUDE.local.md` → `doc_repo_path`
2. Если `doc_repo_path` задан → VISION.md в `<doc_repo_path>/VISION.md`
3. Если `doc_repo_path: null` или отсутствует → VISION.md в корне проекта

**Прочитать:**
- `<vision_path>/VISION.md` — **обязательно**; если отсутствует → СТОП (выше)
- `<doc_repo_path>/PRODUCT.md` (или корень `PRODUCT.md`) — graceful (не блокирует если отсутствует)
- `<doc_repo_path>/DEVLOG.md` последние 10 записей — graceful

## Шаг 1 — North Star Extraction

Из прочитанного VISION.md — **явно назвать** ось или принцип, к которому относится вопрос. Извлекается **в главном агенте** и **передаётся в каждый sub-agent prompt** (Шаг 1.5).

```
📌 North Star anchor: <ось/принцип из VISION.md — цитата или парафраз>
   Ось применима к вопросу потому что: <1-2 предложения>
```

Если VISION.md не содержит прямо применимой оси → написать честно:

```
📌 North Star anchor: VISION.md не содержит прямо применимой оси для этого вопроса.
   Опираюсь на общий контекст: <что именно из VISION.md релевантно косвенно>
```

❌ НЕ натягивать ось через абстрактные связи ради формального заполнения поля.

⛔ **Критично:** sub-agents стартуют с чистого контекста — они **НЕ видят** VISION.md, `doc_repo_path`, историю диалога, если это не передано в их prompt явно. Без North Star в prompt каждый советник даёт **generic** совет, не привязанный к проекту. North Star выжимка (ось + принцип из VISION.md) **обязательна** в каждом prompt.

## Шаг 1.5 — Запуск 7 external советников (parallel)

Запустить **7 sub-agents одним сообщением** (parallel, не последовательно) через Agent tool. Каждый получает prompt по шаблону:

```
Контекст вопроса: <вопрос пользователя дословно>
North Star проекта (из VISION.md): <ось + принцип, извлечённые в Шаге 1 главным агентом>
Релевантный контекст: <2-4 факта из PRODUCT.md/DEVLOG если применимо>

Твоя роль: <роль>
Твой мандат: <мандат роли — см. таблицу>

Дай 2-3 конкретных наблюдения привязанных к ЭТОМУ вопросу + one-line вердикт (✅/⚠️/❌).
⛔ Generic «есть плюсы и минусы» / «интересная идея» не засчитывается.
Верни результат как структурированный текст: наблюдения + вердикт. Это твой возврат, не сообщение человеку.
```

**7 ролей и мандатов:**

| # | Роль | Мандат (передать в prompt) |
|---|---|---|
| 1 | **Ценность** | «Кто конкретно получает пользу и как — через 30 дней? Что изменится у этого человека? **И сколько усилия это стоит получателю (user effort cost): сколько шагов/действий до результата при первом и повторном использовании, есть ли путь в одно действие?** Флагуй если ценность абстрактна, получатель размыт, outcome неизмерим — **ИЛИ если решение оптимизировано под один критерий (безопасность/корректность/скорость), а получатель платит лишними шагами: это нарушение Оси 1 (методология ведёт процесс, не заставляет помнить цепочку) + Actor-burden rule.»** |
| 2 | **North Star** | «Это движет к задекларированной цели проекта (см. North Star выше) или отвлекает? Это лучшее использование ресурса сейчас? Флагуй если не aligned или есть приоритетнее.» |
| 3 | **Горизонт** | «Как изменятся потребности и контекст через 12-24 месяца? Эта ставка умнеет или глупеет со временем? Флагуй если деградирует или закрывает важные будущие возможности.» |
| 4 | **Скептик** *(adversarial)* | «Найди фатальный изъян. Твоя задача — ОПРОВЕРГНУТЬ, не анализировать. Какое ключевое предположение почти наверняка неверно? Нейтральный анализ не засчитывается. Если изъяна правда нет — одной строкой почему.» |
| 5 | **Деятель** | «Первый конкретный шаг в ближайшие 3-7 дней чтобы проверить ключевое предположение? Без проверяемого следующего хода совет бесполезен. Дай конкретный структурный фикс если вердикт ❌/⚠️.» |
| 6 | **Альтернативщик** | «Предложи минимум один принципиально ДРУГОЙ путь достичь той же цели. Чем он лучше/хуже предложенного? Флагуй если выбранный путь — не очевидно лучший из доступных.» |
| 7 | **Complexity tax** | «Это добавление стоит дороже чем даёт? Какую сложность (новые абстракции, поверхность поддержки, когнитивная нагрузка) вносит решение и оправдана ли она ценностью? Флагуй если можно достичь 80% эффекта без этой сложности.» |

### Anti-perfunctory rule

Каждый советник ОБЯЗАН дать конкретное наблюдение, привязанное к этому конкретному вопросу.
❌ «Есть плюсы и минусы» — не засчитывается.
❌ «Интересная идея» — не засчитывается.
✅ «Горизонт: если инструмент устареет через год — эта инвестиция теряет смысл» — конкретно.

### Graceful fallback (Agent tool недоступен)

Если Agent tool **недоступен** в текущем command runtime (spawn вернул ошибку / не поддерживается) → не падать молча:
```
⚠️ /opinion (council 7/7) требует параллельный запуск sub-agents (Agent tool), недоступный сейчас.
   Fallback: симулирую 5 Council-советников в одном контексте (Ценность · North Star · Горизонт · Скептик · Деятель).
   ⚠️ Это solo-симуляция, не structural independence — framing bias не исключён.
   Для true independent council — повтори /opinion в окружении с Agent tool.
```
→ продолжить, симулируя 5 советников «под капотом» (тот же механизм что у inline `[?]`): каждый анализирует вопрос через свою линзу (2-3 наблюдения + one-line вердикт), grid становится `[5/5]` вместо `[7/7]`.

> **Это тот же механизм, что обслуживает inline `[?]`** — лёгкий путь и fallback используют одну симуляцию 5 советников в контексте. Различие только в триггере: `[?]` → симуляция по дизайну (лёгкость); явный `/opinion` → симуляция только если Agent tool недоступен (вынужденный fallback).

## Шаг 2 — Синтез и Committed Verdict

Главный агент читает все 7 возвратов (или 5 при fallback) и формирует **enriched Committed Verdict**.

### Enrichment правила

- **Скептик ❌ → «Что меня беспокоит» ОБЯЗАНА** включить его находку конкретно.
- **Деятель дал структурный фикс → «Условия при которых изменю вердикт»** включает action gap; срабатывает критерий `/plan`-предложения (Шаг 3 B).
- **≥4 из 7 дали ⚠️/❌** (или ≥3 из 5 при fallback) → итоговый вердикт не может быть ✅ без явного обоснования почему перевесил.
- **Альтернативщик предложил лучший путь** → итог должен явно сказать почему держим текущий или переключаемся.

### Committed Verdict — четыре варианта

| Вердикт | Когда использовать |
|---|---|
| ✅ **Поддерживаю** | Вижу чёткий benefit, риски управляемы, соответствует North Star |
| ⚠️ **С оговорками** | Benefit есть, но конкретные условия или риски существенны |
| ❌ **Не поддерживаю** | Противоречит North Star или риски перевешивают benefit |
| 🤷 **Недостаточно контекста** | Ответ принципиально зависит от неизвестного X → запросить X |

### Формат ответа

```
📊 Совет [7/7]: Ценность <s> · North Star <s> · Горизонт <s> · Скептик <s> · Деятель <s> · Альтернативщик <s> · Complexity tax <s>
   Главное расхождение: <роль — что нашла, одна строка>
   (external independent contexts — не симуляция)

## Мнение по: <вопрос кратко>

**Вердикт:** <✅/⚠️/❌/🤷> <одно предложение — главный тезис>

**Почему:**
- <аргумент 1 с конкретикой>
- <аргумент 2>
- <аргумент 3 (опционально)>

**Что меня беспокоит:**
- <конкретная проблема или риск 1>
- <конкретная проблема или риск 2 (опционально)>

**Условия при которых изменю вердикт:**
<что конкретно должно измениться — тогда вердикт изменится на X>
```

При fallback — grid `[5/5]` и пометка `(solo-симуляция — Agent tool недоступен)` вместо `(external independent contexts)`.

**Anti-flattery rule (обязательная секция):** «Что меня беспокоит» **всегда обязательна** — даже при ✅ вердикте. Пустая = команда не завершена.
- ✅ вердикт + пустые беспокойства = red flag (либо нет честного анализа, либо вопрос слишком простой для /opinion).
- Минимум одна конкретная проблема даже при полном согласии.

Секцию «Условия при которых изменю вердикт» — пропустить только для 🤷 (нет основания, не известен контекст).

## Шаг 3 — Optional: Follow-up Proposal

### A. DEVLOG `[opinion:X]`

Если мнение **decision-relevant** (ответ влияет на архитектурное / продуктовое решение):

```
📋 Предлагаю запись в DEVLOG:
[opinion:<slug>] (council 7/7) → <вопрос кратко>: <verdict символ>. <главный тезис ≤60 символов>

(skip — не добавлять; следующее действие = автоматическое подтверждение)
```

Тег остаётся `[opinion:<slug>]` (единый semantic-индикатор, D6 — не фрагментировать grep); council помечается **в тексте** строки: `(council 7/7)`. При fallback — `(solo 5/5)`.

❌ **НЕ предлагать** если вопрос тривиальный или вариативный (style preference, тактический выбор, вопрос предпочтения).

**Implicit confirmation (opt-out):** если пользователь в следующем сообщении не написал явно `skip` / `n` / `нет` / `отменить` — агент добавляет запись в DEVLOG **до начала ответа** на новый запрос. Продолжение работы = согласие.

**Исключения (не добавлять даже без skip):** пользователь явно переключает тему на несвязанный вопрос без какого-либо acknowledgement предложения → treat as implicit skip.

**При добавлении (y или implicit confirmation):**
1. Определить путь к DEVLOG через `doc_repo_path` (как в [/research](research.md))
2. Найти секцию текущей даты `## YYYY-MM-DD` (создать если нет)
3. Добавить строку в конец секции дня

### B. `/plan` предложение (structural fix)

Предлагать `/plan` только при **одновременном** выполнении всех трёх критериев:

1. Вердикт = ❌ или ⚠️
2. Деятель (Шаг 1.5) дал **конкретный структурный фикс** (не «попробуй X», а «добавь правило Y в файл Z» / «обернуть в команду W»)
3. Вопрос касается **архитектуры / механизма / команды / поведения агента** (не style, не тактика, не предпочтение)

Если все три — предложить после DEVLOG блока (A и B могут появляться одновременно):

```
📋 Деятель указал структурный фикс. Предлагаю запустить /plan:
   /plan <краткое описание что исправить>
   Рекомендуемая модель: Default (Sonnet) · effort: High · thinking: ON — план = архитектурный анализ + синтез (deep reasoning). Upgrade до Capable (Opus) если [contract]+threat model / multi-service / class-bug. См. .claude/model-tiers.md § Effort & Thinking.

(skip — не планировать; следующее действие = автоматическое подтверждение)
```

> **Обязательно (трёхмерная рекомендация):** предлагая `/plan` (или любой следующий шаг), агент ВСЕГДА даёт модель в формате `tier · effort · thinking` — не только tier. Effort (слайдер) + Thinking (toggle) пользователь выставляет в UI рядом с выбором модели.

**Implicit confirmation (opt-out):** то же что в пункте A — если пользователь не написал `skip` / `n` / `нет` / `отменить` → продолжение работы = согласие.

❌ **Не предлагать /plan** если: вердикт ✅ или 🤷 · Деятель не дал конкретный структурный фикс · вопрос тактический / стилевой / о предпочтении · критерии не выполнены одновременно.

## Когда использовать (явный `/opinion` vs inline `[?]`)

| | `/opinion` (явный) | inline `[?]` |
|---|---|---|
| Механизм | 7 независимых external sub-agents | 5 советников симулируются в контексте |
| Стоимость / скорость | ~30-60 сек, 7× | ~секунды, 1× |
| Независимость | structural (чистый контекст каждого) | симуляция (один контекст) |
| Когда | важное / необратимое / high-stakes решение, нужна перепроверка с чистого slate | мнение по ходу диалога, лёгкий запрос |

⛔ **Не вызывать `/opinion` (council) внутри другой команды** (`/plan`, `/code` и т.п.): вложенный sub-agent spawn даёт непредсказуемое поведение. Для opinion на шаг плана — заверши команду, затем `/opinion` отдельно. (high-stakes decision-review триггер в `/plan` Шаг -3 рекомендует запуск `/opinion` **после** завершения плана, не вложенно.)

---

## Inline `[?]`

Маркер `[?]` в любом тексте → агент применяет Opinion Protocol **в лёгком режиме**: 5 советников симулируются в одном контексте (без external spawn), North Star anchor + committed verdict. Для high-stakes используй явный `/opinion` (полный council-7).

## Сравнение с `/research`

| | `/opinion` | `/research` |
|---|---|---|
| Когда | «Что ты думаешь о X?» | «Исследуй факты о X» |
| Источник | VISION.md + reasoning | WebSearch + источники |
| Output | Позиция агента | Факты + verdict |
| DEVLOG | `[opinion:X]` (optional, decision-relevant) | `[research:X]` (обязателен) |
