# /opinion — Честное мнение с North Star

> **Цель:** получить честное, структурированное мнение агента по конкретному вопросу, заземлённое в North Star проекта. Команда для explicit opinion request — не замена свободному чату, а формализованный путь когда мнение важно для решения.

**Critical Friend Protocol:** не поддакивать, не хеджировать. Чёткая позиция с обоснованием в контексте VISION.md.

**Council Protocol:** при каждом вызове автоматически запускает 5 советников «под капотом» (Ценность · North Star · Горизонт · Скептик · Деятель); агент обогащает Committed Verdict их инсайтами. Вывод: 2 строки сигнала; по запросу «разверни совет» — полный вывод.

**Два режима:**
- **`/opinion`** (solo) — 5 советников **симулируются** в одном контексте агента. Быстро (~секунды). Дефолт для большинства вопросов.
- **`/opinion+`** (council) — те же 5 советников + 2 дополнительных (Альтернативщик · Complexity tax) запускаются как **7 независимых external sub-agents**, каждый со своим чистым контекстом → structural independence (нет framing bias накопленного в диалоге). Медленно (~30-60 сек). **Только для необратимых / высоких ставок.** См. секцию «## Режим /opinion+».

---

## Рекомендуемая модель

**Default tier:** **Default** (Sonnet) — reasoning + North Star extraction + committed verdict достаточно для большинства вопросов.
**Upgrade to Capable if:** Вопрос затрагивает несколько сервисов с конфликтующими сигналами; глубокий стратегический trade-off требует cross-domain analysis.
**Downgrade to Fast:** ❌ НЕ рекомендуется — North Star extraction требует reasoning.
**Mid-task escalation:** нет — opinion-flow не требует reassessment mid-task.
**Pre-flight model check:** да — спроси пользователя о модели при старте, или используй ранее подтверждённую в сессии.

## Шаг 1.5 — Council Protocol

Запускается автоматически при каждом вызове `/opinion`.

### Внутренний анализ (не выводить развёрнуто)

Запустить 5 советников «под капотом». Каждый анализирует вопрос через свою линзу (2-3 наблюдения + one-line вердикт). Полный вывод каждого советника — только если пользователь явно пишет «разверни совет».

**1. Ценность**
*Мандат:* «Кто конкретно получает пользу и как — через 30 дней? Что изменится у этого человека?»
Флагует если: ценность абстрактна, получатель размыт, outcome неизмерим.

**2. North Star**
*Мандат:* «Это движет к задекларированной цели проекта или отвлекает? Это лучшее использование ресурса сейчас?»
Читает: VISION.md (загружен в Шаге 0). Флагует если: не aligned с VISION, есть более приоритетная альтернатива.

**3. Горизонт**
*Мандат:* «Как изменятся потребности, контекст, условия через 12-24 месяца? Эта ставка умнеет или глупеет со временем?»
Флагует если: решение хорошо сейчас но деградирует, или закрывает важные будущие возможности.

**4. Скептик** *(adversarial mandate)*
*Мандат:* «Найди фатальный изъян. Твоя задача — опровергнуть, не анализировать. Какое ключевое предположение здесь почти наверняка неверно?»
⛔ Нейтральный анализ не засчитывается. Только опровержение.
Если изъяна действительно нет → написать одной строкой почему не нашёл.

**5. Деятель**
*Мандат:* «Первый конкретный шаг — что делать в ближайшие 3-7 дней чтобы проверить ключевое предположение? Без этого шага совет бесполезен.»
Флагует если: нет action vector, решение остаётся в воздухе без проверяемого следующего хода.

### Anti-perfunctory rule

Каждый советник ОБЯЗАН дать конкретное наблюдение, привязанное к этому конкретному вопросу.
❌ «Есть плюсы и минусы» — не засчитывается.
❌ «Интересная идея» — не засчитывается.
✅ «Горизонт: если инструмент устареет через год — эта инвестиция теряет смысл» — конкретно.

### Вывод совета пользователю (2 строки максимум)

```
📊 Совет [5/5]: Ценность ⚠️ · North Star ✅ · Горизонт ✅ · Скептик ❌ · Деятель ⚠️
   Главное расхождение: Скептик — [одна строка — что именно нашёл]
```

Если все советники единодушны:
```
📊 Совет [5/5]: единодушно ⚠️ — [главный общий тезис одной строкой]
```

Если Скептик не нашёл изъяна:
```
📊 Совет [5/5]: Ценность ⚠️ · North Star ✅ · Горизонт ✅ · Скептик ✅ (изъяна не найдено) · Деятель ⚠️
```

### Enrichment правила для Шага 2

Инсайты совета обязательно учитываются в Committed Verdict:
- Скептик ❌ → **«Что меня беспокоит»** ОБЯЗАНА включить его находку конкретно
- Деятель ⚠️ или ❌ → **«Условия при которых изменю вердикт»** ОБЯЗАНА включить action gap
- ≥3 советников ⚠️/❌ → Committed Verdict не может быть ✅ без явного обоснования почему перевесили
- По запросу «разверни совет» → вывести полный анализ каждого советника

## Шаг 2 — Анализ и Committed Verdict

**Council (Шаг 1.5) всегда запущен:** инсайты советников обязательно отражаются в вердикте — см. Enrichment правила в Шаге 1.5.

Сформулировать честную позицию. **Четыре варианта:**

| Вердикт | Когда использовать |
|---|---|
| ✅ **Поддерживаю** | Вижу чёткий benefit, риски управляемы, соответствует North Star |
| ⚠️ **С оговорками** | Benefit есть, но конкретные условия или риски существенны |
| ❌ **Не поддерживаю** | Противоречит North Star или риски перевешивают benefit |
| 🤷 **Недостаточно контекста** | Ответ принципиально зависит от неизвестного X → запросить X |

**Формат ответа:**

```
📊 Совет [5/5]: <grid из Шага 1.5 — ОБЯЗАТЕЛЬНО первая строка>
   Главное расхождение: <finding или «нет расхождений»>

## Мнение по: <вопрос кратко>

**Вердикт:** <✅/⚠️/❌/🤷> <одно предложение — главный тезис>

**Почему:**
- <аргумент 1 с конкретикой>
- <аргумент 2>
- <аргумент 3 (опционально)>

**Что меня беспокоит:**
- <конкретная проблема или риск 1>
- <конкретная проблема или риск 2 (опционально)>

**Условия при которых изменю вердикт:**
<что конкретно должно измениться — тогда вердикт изменится на X>
```

**Anti-flattery rule (обязательная секция):** «Что меня беспокоит» **всегда обязательна** — даже при ✅ вердикте. Пустая = команда не завершена.
- ✅ вердикт + пустые беспокойства = red flag (либо нет честного анализа, либо вопрос слишком простой для /opinion).
- Минимум одна конкретная проблема даже при полном согласии.

Секцию «Условия при которых изменю вердикт» — пропустить только для 🤷 (нет основания, не известен контекст).

## Шаг 3 — Optional: Follow-up Proposal

### A. DEVLOG `[opinion:X]`

Если мнение **decision-relevant** (ответ влияет на архитектурное / продуктовое решение):

```
📋 Предлагаю запись в DEVLOG:
[opinion:<slug>] → <вопрос кратко>: <verdict символ>. <главный тезис ≤60 символов>

(skip — не добавлять; следующее действие = автоматическое подтверждение)
```

❌ **НЕ предлагать** если вопрос тривиальный или вариативный (style preference, тактический выбор, вопрос предпочтения).

**Implicit confirmation (opt-out):** если пользователь в следующем сообщении не написал явно `skip` / `n` / `нет` / `отменить` — агент добавляет запись в DEVLOG **до начала ответа** на новый запрос. Продолжение работы = согласие.

**Исключения (не добавлять даже без skip):** пользователь явно переключает тему на несвязанный вопрос без какого-либо acknowledgement предложения → treat as implicit skip.

**При добавлении (y или implicit confirmation):**
1. Определить путь к DEVLOG через `doc_repo_path` (как в [/research](research.md))
2. Найти секцию текущей даты `## YYYY-MM-DD` (создать если нет)
3. Добавить строку в конец секции дня

### B. `/plan` предложение (structural fix)

Предлагать `/plan` только при **одновременном** выполнении всех трёх критериев:

1. Вердикт = ❌ или ⚠️
2. Деятель (Шаг 1.5) дал **конкретный структурный фикс** (не «попробуй X», а «добавь правило Y в файл Z» / «обернуть в команду W»)
3. Вопрос касается **архитектуры / механизма / команды / поведения агента** (не style, не тактика, не предпочтение)

Если все три — предложить после DEVLOG блока (A и B могут появляться одновременно):

```
📋 Деятель указал структурный фикс. Предлагаю запустить /plan:
   /plan <краткое описание что исправить>

(skip — не планировать; следующее действие = автоматическое подтверждение)
```

**Implicit confirmation (opt-out):** то же что в пункте A — если пользователь не написал `skip` / `n` / `нет` / `отменить` → продолжение работы = согласие.

❌ **Не предлагать /plan** если: вердикт ✅ или 🤷 · Деятель не дал конкретный структурный фикс · вопрос тактический / стилевой / о предпочтении · критерии не выполнены одновременно.

## Режим /opinion+

> **Что это:** high-stakes вариант. Те же 5 Council-советников **+ 2 дополнительных линзы** запускаются как **7 независимых external sub-agents**, каждый со своим чистым контекстом. Главный агент синтезирует их вердикты.
>
> **Зачем external, а не усиленный prompt в solo:** solo Council симулирует советников **в одном контексте** — агент, который уже «продал» идею в первых сообщениях диалога, защищает её и в роли Скептика (framing bias накопленного контекста). External sub-agent стартует с **чистого slate** → structural independence, не симуляция. Это L4-усиление (отдельный контекст), не L1 (другая инструкция в том же контексте).

### Когда использовать (строгий триггер)

⛔ `/opinion+` — НЕ для рутины. 7 sub-agents = ~30-60 сек + 7× стоимость. Применять **только** при хотя бы одном:
- **Необратимое решение** — удаление команды/фичи, breaking change схемы, миграция данных, major version bump.
- **Высокая цена ошибки** — security-решение, контракт с внешним миром, архитектурная развилка влияющая на ≥3 компонента.
- **Framing-bias риск** — вопрос возник в конце длинного диалога, где агент уже занял позицию; нужна независимая перепроверка с чистого контекста.

Для всего остального → `/opinion` (solo). Если сомневаешься «+ или нет» → solo: дешевле, а при недовольстве можно эскалировать до `+`.

⛔ **Не вызывать `/opinion+` внутри другой команды** (`/plan`, `/code` и т.п.): вложенный sub-agent spawn даёт непредсказуемое поведение. Для opinion на шаг плана — заверши команду, затем `/opinion+` отдельно.

### Шаг +0 — Pre-flight (наследует Шаг 0)

Выполнить Шаг 0 (VISION.md обязателен) и Шаг 1 (North Star Extraction) **в главном агенте**. North Star anchor, извлечённый здесь, **передаётся в каждый sub-agent prompt** — см. Шаг +1.

⛔ **Критично:** sub-agents стартуют с чистого контекста — они **НЕ видят** VISION.md, `doc_repo_path`, историю диалога, если это не передано в их prompt явно. Без North Star в prompt каждый советник даёт **generic** совет, не привязанный к проекту. North Star выжимка (ось + принцип из VISION.md) **обязательна** в каждом prompt.

### Шаг +1 — Запуск 7 external советников (parallel)

Запустить **7 sub-agents одним сообщением** (parallel, не последовательно) через Agent tool. Каждый получает prompt по шаблону:

```
Контекст вопроса: <вопрос пользователя дословно>
North Star проекта (из VISION.md): <ось + принцип, извлечённые в Шаге +1 главным агентом>
Релевантный контекст: <2-4 факта из PRODUCT.md/DEVLOG если применимо>

Твоя роль: <роль>
Твой мандат: <мандат роли — см. таблицу>

Дай 2-3 конкретных наблюдения привязанных к ЭТОМУ вопросу + one-line вердикт (✅/⚠️/❌).
⛔ Generic «есть плюсы и минусы» / «интересная идея» не засчитывается.
Верни результат как структурированный текст: наблюдения + вердикт. Это твой возврат, не сообщение человеку.
```

**7 ролей и мандатов:**

| # | Роль | Мандат (передать в prompt) |
|---|---|---|
| 1 | **Ценность** | «Кто конкретно получает пользу и как — через 30 дней? Что изменится у этого человека? Флагуй если ценность абстрактна, получатель размыт, outcome неизмерим.» |
| 2 | **North Star** | «Это движет к задекларированной цели проекта (см. North Star выше) или отвлекает? Это лучшее использование ресурса сейчас? Флагуй если не aligned или есть приоритетнее.» |
| 3 | **Горизонт** | «Как изменятся потребности и контекст через 12-24 месяца? Эта ставка умнеет или глупеет со временем? Флагуй если деградирует или закрывает важные будущие возможности.» |
| 4 | **Скептик** *(adversarial)* | «Найди фатальный изъян. Твоя задача — ОПРОВЕРГНУТЬ, не анализировать. Какое ключевое предположение почти наверняка неверно? Нейтральный анализ не засчитывается. Если изъяна правда нет — одной строкой почему.» |
| 5 | **Деятель** | «Первый конкретный шаг в ближайшие 3-7 дней чтобы проверить ключевое предположение? Без проверяемого следующего хода совет бесполезен. Дай конкретный структурный фикс если вердикт ❌/⚠️.» |
| 6 | **Альтернативщик** | «Предложи минимум один принципиально ДРУГОЙ путь достичь той же цели. Чем он лучше/хуже предложенного? Флагуй если выбранный путь — не очевидно лучший из доступных.» |
| 7 | **Complexity tax** | «Это добавление стоит дороже чем даёт? Какую сложность (новые абстракции, поверхность поддержки, когнитивная нагрузка) вносит решение и оправдана ли она ценностью? Флагуй если можно достичь 80% эффекта без этой сложности.» |

**Graceful fallback:** если Agent tool **недоступен** в текущем command runtime (spawn вернул ошибку / не поддерживается) → не падать молча:
```
⚠️ /opinion+ требует параллельный запуск sub-agents (Agent tool), недоступный сейчас.
   Fallback: выполняю solo /opinion (5 советников симулируются в одном контексте).
   Для true structural independence — повтори /opinion+ в окружении с Agent tool.
```
→ продолжить как обычный `/opinion` (Шаг 1.5 Council Protocol internal).

### Шаг +2 — Синтез

Главный агент читает все 7 возвратов и формирует **enriched Committed Verdict**:

- **Grid (8 строк):** вывести вердикт каждого из 7 + итоговый синтез.
- **Скептик ❌ → «Что меня беспокоит» ОБЯЗАНА** включить его находку конкретно.
- **Деятель дал структурный фикс → «Условия при которых изменю вердикт»** включает action gap; срабатывает критерий `/plan`-предложения (Шаг 3 B).
- **≥4 из 7 дали ⚠️/❌** → итоговый вердикт не может быть ✅ без явного обоснования почему перевесил.
- **Альтернативщик предложил лучший путь** → итог должен явно сказать почему держим текущий или переключаемся.

Формат вывода — как в Шаге 2 (Committed Verdict), но grid содержит 7 строк вместо 5:

```
📊 Совет [7/7]: Ценность <s> · North Star <s> · Горизонт <s> · Скептик <s> · Деятель <s> · Альтернативщик <s> · Complexity tax <s>
   Главное расхождение: <роль — что нашла, одна строка>
   (external independent contexts — не симуляция)

## Мнение по: <вопрос кратко>
<далее формат Шага 2: Вердикт / Почему / Что меня беспокоит / Условия изменения>
```

### Шаг +3 — Follow-up

Идентичен Шагу 3 (DEVLOG `[opinion:X]` + опциональное `/plan`-предложение). Тег остаётся `[opinion:<slug>]` (единый semantic-индикатор, D6 — не фрагментировать grep); режим council помечается **в тексте** строки: `[opinion:<slug>] (council 7/7) → ...`.

## Inline `[?]`

Маркер `[?]` в любом тексте → агент применяет Opinion Protocol (solo) без отдельного `/opinion`. Лёгкий путь; для high-stakes используй явный `/opinion+`.

## Сравнение с `/research`

| | `/opinion` | `/research` |
|---|---|---|
| Когда | «Что ты думаешь о X?» | «Исследуй факты о X» |
| Источник | VISION.md + reasoning | WebSearch + источники |
| Output | Позиция агента | Факты + verdict |
| DEVLOG | `[opinion:X]` (optional, decision-relevant) | `[research:X]` (обязателен) |
